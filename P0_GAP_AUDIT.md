# ORBIT P0 기준 차이점 점검

- 점검일: 2026-07-13
- 대상: 개인 프로토타입 `pangyo-festival-app`
- 기준: ORBIT ADR-0001, PROJECT_GUIDE, DESIGN_SYSTEM, API_GUIDE, DATABASE_GUIDE, CODE_GUIDE, SECURITY_GUIDE, QA_GUIDE
- 범위: 이 문서는 개인 저장소의 UI 프로토타입 점검 기록이다. ORBIT 공용 저장소, 공용 Supabase, 기존 문서는 수정하지 않는다.

## 결론

현재 앱은 **부스 지도와 화면 흐름을 검증하는 개인 UI 프로토타입**으로는 사용할 수 있다. 하지만 로그인, NFC 적립, 운영자 권한, 관리자 기능, 스탬프 저장은 브라우저 `localStorage`와 데모 데이터에 의존한다. 따라서 학생 개인정보나 실제 보상과 연결되는 P0 운영 서비스로 사용하면 안 된다.

P0 연동 전에는 ORBIT의 인증, 서버 API, 행사 범위 DB, 서명 NFC 토큰, 감사 로그가 필요하다.

## 2026-07-13 개인 프로토타입 반영

이 점검 이후 개인 앱에 다음 UI 작업을 반영했다.

- 홈/부스/NFC/패스/내 정보의 5개 학생 화면
- 행사·동아리·교실·운영 시간·운영 상태를 포함한 부스 데이터 어댑터
- `preparing/open/crowded/paused/closed` 상태 배지
- 공지, 비상 모드, NFC 실패 시 수동 승인 안내
- NFC 성공·중복·운영 중지·미등록 태그 결과 화면
- ORBIT 파란색 토큰, 48px 터치 영역, 8px 반경, 320px 대응

이 변경은 UI와 로컬 데모 데이터에만 적용됐다. 아래의 인증·서버·DB·감사 관련 차이는 그대로 남아 있다.

## P0 출시를 막는 차이

### 1. 인증과 권한

- Google 버튼은 실제 OAuth/OIDC 인증이 아니라 고정 데모 프로필을 반환한다.
- 관리자 모드도 고정 관리자 계정으로 진입한다.
- 행사별 `event_roles` 권한 확인이 없다.
- 브라우저 데이터만 바꾸면 사용자, 관리자, 기록을 조작할 수 있다.

필요한 교체: 서버 세션 기반 학교 계정 인증, `eventId` 범위 역할 확인, 담당 부스 범위 검사.

### 2. NFC 스탬프

- `?nfc=...` URL 파라미터와 화면 버튼이 태그 ID를 직접 전달한다.
- NFC 태그 서명, 행사/부스 일치, 만료, nonce, 서버 검증이 없다.
- 적립과 중복 방지는 `localStorage` 배열에서만 처리한다.
- NFC 실패 시 운영자 수동 승인 API와 감사 기록이 없다.

필요한 교체: 서명된 NFC 토큰을 서버 `POST /api/v1/events/{eventId}/stamps`로 전달하고, `Idempotency-Key`, DB transaction, 활성 스탬프 unique 제약으로 처리.

### 3. 행사와 데이터

- 앱 데이터에 `event_id`가 없다.
- 부스 상태, 운영 시간, 공지, 비상 모드가 P0 계약대로 연결되어 있지 않다.
- 사용자/스탬프/리뷰/교환 데이터가 한 브라우저의 `localStorage`에만 저장된다.

필요한 교체: ORBIT 기준선 migration과 API가 준비된 뒤 서버 원본 데이터를 조회하도록 교체.

### 4. 운영 화면과 감사

- 운영자와 관리자 권한이 분리되지 않았다.
- 부스 추가/삭제, NFC 값 변경, 리뷰 삭제, 교환 처리에 서버 권한, 처리자, 사유, 감사 로그가 없다.
- 다른 부스 운영자가 접근할 수 없게 막는 서버 검증이 없다.

필요한 교체: 담당 부스 운영자 전용 상태 변경/수동 승인 화면과 관리자 전용 공지·비상 모드 화면을 분리.

### 5. P0 범위 정리

- 리뷰는 P1 기능이다.
- 교환 처리와 관리자용 상품 관련 UI는 P1 기능이다.
- 현재 앱에 이 기능들이 함께 보이므로 P0 버전에서는 숨기거나 별도 실험 화면으로 격리해야 한다.

## 디자인 계약 차이

| 기준 | 개인 앱 현재값 | 이후 조치 |
| --- | --- | --- |
| 주요 색상 | 코랄/빨강 `#ff4f64` | ORBIT 통합 후보는 기본색 `#2563EB`로 별도 테마 적용 |
| 카드 반경 | 일부 14~26px | P0 공통 컴포넌트는 기본 8px 기준으로 정리 |
| 하단 메뉴 | 지도 중심 메뉴 구조 | 홈/부스/스캔/패스/내 정보, 최대 5개로 재구성 |
| 상태 | 방문 여부 중심 | `preparing/open/crowded/paused/closed` 상태 배지 추가 |
| 안내 | 개별 화면 문구 | 공지·비상 모드 배너를 핵심 화면에서 읽을 수 있게 추가 |

지도 자체는 P1 범위이므로 현재 지도 완성도는 유지하되, P0 부스 목록·검색·층/교실 위치와 같은 데이터를 먼저 안정화한다.

## 개인 저장소에서 진행 가능한 순서

1. 기존 개인 앱의 스탬프, 관리자, 리뷰 화면에 "데모" 경계를 명확히 하고 P0 전용 화면 흐름을 별도 구성한다.
2. 부스 데이터 모양을 `eventId`, `clubName`, `floor`, `room`, `status`, `opensAt`, `closesAt` 중심의 읽기 전용 어댑터로 분리한다.
3. 부스 상태 배지, 공지 배너, 비상 모드, NFC 실패 시 수동 승인 안내 UI를 개인 프로토타입에 추가한다.
4. 로그인과 NFC는 실제 저장을 하지 않는 목업 상태로 유지하고, 서버 API가 확정된 뒤 교체한다.
5. Android/iOS/320px 기준으로 지도 조작, 하단 목록, 검색, 뒤로 가기, 모션 감소를 점검한다.
